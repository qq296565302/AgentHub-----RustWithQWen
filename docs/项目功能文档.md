## 项目概述

AgentHub 是一个基于 Rust 构建的本地 AI 执行引擎，具有以下特点：

- **安全性优先**：内置多层安全管道，防止 PII 泄露、Prompt Injection 等风险
- **Skill 驱动**：通过可扩展的 Skill 系统实现代码解释、测试生成等功能
- **多 LLM 支持**：支持 OpenAI、Ollama 等多种 LLM 提供商，可动态切换
- **本地优先**：所有数据处理在本地完成，保护隐私
- **双模式运行**：支持 REPL 交互模式和 HTTP API 服务模式

## 核心功能

### 1. REPL 交互模式

REPL（Read-Eval-Print Loop）提供了一个命令行交互界面，用户可以直接输入文本与 AI 对话，或使用斜杠命令执行操作。

#### 使用方法

启动 REPL 后，直接输入文本即可与 AI 对话：

```
agenthub> 你好，请介绍一下你自己
```

#### 可用命令

| 命令 | 说明 | 示例 |
|------|------|------|
| `/skills` | 列出所有可用的 Skill | `/skills` |
| `/explain` | 解释代码 | `/explain src/main.rs` |
| `/explain` | 解释特定行 | `/explain src/main.rs:42` |
| `/explain` | 解释行范围 | `/explain src/main.rs:10:20` |
| `/explain` | 解释函数 | `/explain src/main.rs::main` |
| `/test` | 为函数生成测试 | `/test src/main.rs main` |
| `/config` | 显示当前配置 | `/config` |
| `/provider` | 显示当前 LLM 提供商 | `/provider` |
| `/provider` | 切换 LLM 提供商 | `/provider mini` |
| `/providers` | 列出所有 LLM 提供商 | `/providers` |
| `/conv new` | 创建新对话 | `/conv new my_chat` |
| `/conv list` | 列出所有对话 | `/conv list` |
| `/conv switch` | 切换对话 | `/conv switch my_chat` |
| `/conv delete` | 删除对话 | `/conv delete my_chat` |
| `/conv clear` | 清空当前对话历史 | `/conv clear` |
| `/audit` | 查询审计日志 | `/audit` |
| `/clear` | 清空对话历史 | `/clear` |
| `/help` | 显示帮助 | `/help` |
| `/exit` | 退出 REPL | `/exit` |

#### 自然语言命令

系统支持通过自然语言直接执行命令，无需输入斜杠前缀。当用户输入的内容被 LLM 解析为命令意图时（置信度 >= 0.7），系统会优先执行命令而非进行对话。

| 自然语言输入 | 等效命令 | 说明 |
|-------------|---------|------|
| "解释一下 main.rs 文件" | `/explain src/main.rs` | 解释代码 |
| "解释 main.rs 的第 42 行" | `/explain src/main.rs:42` | 解释特定行 |
| "解释 main.rs 里的 main 函数" | `/explain src/main.rs::main` | 解释函数 |
| "给 main.rs 的 main 函数写个测试" | `/test src/main.rs main` | 生成测试 |
| "看看有哪些技能" | `/skills` | 列出 Skills |
| "显示当前配置" | `/config` | 显示配置 |
| "切换到 mini 模型" | `/provider mini` | 切换提供商 |
| "列出所有模型" | `/providers` | 列出提供商 |
| "新建一个对话" | `/conv new` | 创建对话 |
| "列出所有对话" | `/conv list` | 列出对话 |
| "切换到对话 xxx" | `/conv switch xxx` | 切换对话 |
| "删除对话 xxx" | `/conv delete xxx` | 删除对话 |
| "清空对话" | `/conv clear` | 清空对话 |
| "查看审计日志" | `/audit` | 查询审计日志 |
| "清空历史" | `/clear` | 清空历史 |
| "显示帮助" | `/help` | 显示帮助 |
| "退出" | `/exit` | 退出 REPL |

#### 实现逻辑

- **斜杠命令解析**：[commands.rs](file:///d:/KaKaRoot/AgentHub智能体中枢RustWithQWen/src/api/cli/commands.rs) 使用字符串解析，以 `/` 前缀区分命令和普通聊天消息
- **自然语言命令解析**：[nl_command_parser.rs](file:///d:/KaKaRoot/AgentHub智能体中枢RustWithQWen/src/api/cli/nl_command_parser.rs) 通过 LLM 解析用户输入的自然语言，判断是否为命令意图
  - 系统提示词中定义了所有可用命令和 Skills
  - LLM 返回 JSON 格式的命令解析结果（包含命令类型、参数、置信度）
  - 置信度 >= 0.7 时执行命令，否则作为普通对话处理
- **命令优先级**：自然语言命令解析优先级高于普通对话，确保命令意图被优先执行
- **无递归调用**：通过 `handle_chat`（含命令解析）和 `do_chat`（纯对话）分离，避免异步递归问题
- **历史记录**：使用 `rustyline` 库实现命令历史
- **Markdown 渲染**：LLM 返回的 Markdown 格式内容通过自定义渲染器在终端中格式化显示

### 2. HTTP API 服务

提供 RESTful API 接口，供前端或其他服务调用。

#### 可用端点

| 端点 | 方法 | 说明 |
|------|------|------|
| `/health` | GET | 健康检查 |
| `/api/explain` | POST | 解释代码 |
| `/api/test` | POST | 生成测试 |
| `/api/skills` | GET | 列出 Skills |

#### 请求示例

```bash
# 健康检查
curl http://localhost:8080/health

# 解释代码
curl -X POST http://localhost:8080/api/explain \
  -H "Content-Type: application/json" \
  -d '{"file_path": "src/main.rs", "function_name": "main"}'

# 生成测试
curl -X POST http://localhost:8080/api/test \
  -H "Content-Type: application/json" \
  -d '{"file_path": "src/main.rs", "function_name": "main"}'

# 列出 Skills
curl http://localhost:8080/api/skills
```

#### 实现逻辑

- **框架**：使用 `axum` 作为 HTTP 框架
- **状态管理**：通过 `AppState` 共享配置、LLM 客户端、Skill 注册表和安全管道
- **安全集成**：每个端点都经过速率限制、输入检查和输出消毒

### 3. 多 LLM 提供商支持

系统支持配置多个 LLM 服务，并可在运行时动态切换。

#### 支持的提供商

| 类型 | 说明 | 配置项 |
|------|------|--------|
| `openai` | 兼容 OpenAI API 的服务 | `api_key`, `model`, `api_endpoint` |
| `ollama` | 本地 Ollama 服务 | `model`, `api_endpoint` |
| 其他 | 自动降级为 Mock | - |

#### 配置示例

```yaml
llm:
  default_provider: "mini"
  providers:
    - name: "mini"
      nickname: "简单模型"
      description: "用于快速回复和简单任务"
      type: openai
      api_key: "your-api-key"
      model: "DeepSeek-R1-0528-Qwen3-8B"
      api_endpoint: "https://api.edgefn.net/v1"
      temperature: 0.7
      timeout_secs: 120

    - name: "local"
      nickname: "本地模型"
      type: ollama
      model: "qwen:4b"
      api_endpoint: "http://localhost:11434"
```

#### 实现逻辑

- **架构**：[MultiLLMClient](file:///d:/KaKaRoot/AgentHub智能体中枢RustWithQWen/src/llm/multi_client.rs) 维护一个客户端映射表，通过 `current_provider` 追踪当前使用的提供商
- **动态切换**：`switch_provider()` 方法更新当前提供商，无需重启服务
- **Trait 抽象**：所有 LLM 客户端实现 `LLMClient` trait，保证接口一致性
- **线程安全**：使用 `RwLock` 实现并发安全的读写操作

### 4. 对话上下文管理

系统维护多轮对话历史，自动管理 Token 窗口，确保上下文在模型限制内。

#### 功能特性

- **多对话支持**：可同时维护多个对话会话
- **Token 计数**：使用 `tiktoken-rs` 精确计算 Token 数量
- **自动截断**：当 Token 数超过限制时，自动移除最早的消息
- **系统提示**：支持为每个对话设置独立的系统提示

#### 使用方法

```
agenthub> /conv new          # 创建新对话
agenthub> /conv list         # 查看所有对话
agenthub> /conv switch conv_xxx  # 切换对话
agenthub> /conv clear        # 清空当前对话
```

#### 实现逻辑

- **数据结构**：[ConversationContext](file:///d:/KaKaRoot/AgentHub智能体中枢RustWithQWen/src/prompt/conversation.rs) 存储消息列表、系统提示、Token 计数
- **窗口管理**：`trim_to_context_window()` 方法从消息列表头部移除旧消息，直到 Token 数在限制内
- **上下文管理器**：[ContextManager](file:///d:/KaKaRoot/AgentHub智能体中枢RustWithQWen/src/prompt/context_manager.rs) 提供 Token 计数和文本截断功能

### 5. Skill 系统

Skill 是可插拔的功能模块，通过统一的接口实现代码解释、测试生成等能力。

#### 内置 Skills

| Skill 名称 | 版本 | 说明 |
|-----------|------|------|
| `code.explainer` | 0.1.0 | 解释代码逻辑及潜在风险 |
| `code.test.generator` | 0.1.0 | 为函数生成单元测试 |

#### Skill 接口

```rust
#[async_trait]
pub trait SkillExecutor: Send + Sync {
    async fn execute(
        &self,
        params: serde_json::Value,
        context: &ExecutionContext,
    ) -> Result<SkillResult>;
}
```

#### 实现逻辑

- **注册表**：[SkillRegistry](file:///d:/KaKaRoot/AgentHub智能体中枢RustWithQWen/src/skill/mod.rs) 使用 HashMap 管理所有 Skill
- **执行上下文**：`ExecutionContext` 提供用户 ID 和工作目录
- **结果返回**：`SkillResult` 包含输出、创建的文件列表和警告信息

### 6. 代码解析器

基于 Tree-sitter 的多语言代码解析器，支持 AST 级别的代码分析。

#### 支持的语言

| 语言 | Tree-sitter 包 |
|------|---------------|
| Rust | `tree-sitter-rust` |
| Python | `tree-sitter-python` |
| JavaScript | `tree-sitter-javascript` |
| TypeScript | `tree-sitter-typescript` |

#### 功能

- **语言注册**：[LanguageRegistry](file:///d:/KaKaRoot/AgentHub智能体中枢RustWithQWen/src/parser/language_registry.rs) 动态加载语言解析器
- **AST 工具**：[ast_utils](file:///d:/KaKaRoot/AgentHub智能体中枢RustWithQWen/src/parser/ast_utils.rs) 提供通用的 AST 操作
- **语言特定解析器**：每种语言有独立的解析实现

#### 实现逻辑

- **Tree-sitter 集成**：使用 Tree-sitter 解析代码为抽象语法树
- **按需加载**：通过 Cargo features 控制语言包编译
- **扩展性**：新增语言只需添加对应的 Tree-sitter 包和解析器

### 7. 安全管道

多层安全防护系统，对所有输入输出进行检查和处理。

#### 安全层级

| 层级 | 模块 | 功能 | 配置项 |
|------|------|------|--------|
| 1 | **InputValidator** | 输入大小限制、空输入检查 | `max_input_length` |
| 2 | **PromptInjectionGuard** | Prompt Injection 检测 | `prompt_injection.enabled` |
| 3 | **PIIRedactionGuard** | PII 检测和脱敏 | `pii_detection.*` |
| 4 | **UnicodeSecurityGuard** | Unicode 同形字、零宽字符检测 | `prompt_injection.check_unicode` |
| 5 | **FileAccessGuard** | 文件访问控制 | `file_access.*` |
| 6 | **NetworkGuard** | 网络访问控制 | `network.*` |
| 7 | **OutputSanitizer** | 输出消毒 | `output_sanitizer.*` |
| 8 | **RateLimiter** | 速率限制 | `rate_limit.*` |

#### 安全分级

```rust
pub enum SecurityLevel {
    Pass,           // 通过
    Warn(String),   // 警告
    Block(String),  // 阻断
}
```

#### 各模块详细说明

**InputValidator** - [input_validator.rs](file:///d:/KaKaRoot/AgentHub智能体中枢RustWithQWen/src/guardrails/input_validator.rs)
- 检查输入长度是否超过 `max_input_length`
- 拒绝空输入

**PromptInjectionGuard** - [prompt_injection.rs](file:///d:/KaKaRoot/AgentHub智能体中枢RustWithQWen/src/guardrails/prompt_injection.rs)
- 使用正则匹配已知注入模式：
  - `ignore previous instructions`
  - `you are now acting as`
  - `system:`
  - `<s>` / `</s>` 标签
  - `[INST]` 标签
  - `DAN mode`
  - `jailbreak`
- 检测 Unicode 同形字攻击（西里尔字母伪装）
- 检测零宽字符隐藏攻击

**PIIRedactionGuard** - [pii_detector.rs](file:///d:/KaKaRoot/AgentHub智能体中枢RustWithQWen/src/guardrails/pii_detector.rs)
- 检测类型：邮箱、手机号（中国大陆）、身份证号、IP 地址、API 密钥
- 输入检测：发现 PII 时阻断请求
- 输出脱敏：自动替换为 `[EMAIL_REDACTED]` 等标记

**FileAccessGuard** - [file_access.rs](file:///d:/KaKaRoot/AgentHub智能体中枢RustWithQWen/src/guardrails/file_access.rs)
- 路径遍历检测（`../`）
- 敏感文件保护（`.env`, `.git/`, `id_rsa`, `.ssh/` 等）
- 符号链接防护（防止链接到工作目录外）
- 工作目录限制

**NetworkGuard** - [network_guard.rs](file:///d:/KaKaRoot/AgentHub智能体中枢RustWithQWen/src/guardrails/network_guard.rs)
- 内网 IP 拦截（10.x, 172.16-31.x, 192.168.x, 127.x）
- 域名白名单机制
- 外网访问开关控制

**OutputSanitizer** - [output_sanitizer.rs](file:///d:/KaKaRoot/AgentHub智能体中枢RustWithQWen/src/guardrails/output_sanitizer.rs)
- 控制字符过滤（保留换行、回车、制表符）
- PII 脱敏处理

**RateLimiter** - [rate_limiter.rs](file:///d:/KaKaRoot/AgentHub智能体中枢RustWithQWen/src/guardrails/rate_limiter.rs)
- 滑动窗口算法
- 每分钟请求数限制
- 突发请求限制

#### 实现逻辑

- **管道模式**：[SecurityPipeline](file:///d:/KaKaRoot/AgentHub智能体中枢RustWithQWen/src/guardrails/mod.rs) 按顺序执行各层检查
- **快速失败**：任何一层阻断即返回，不再继续后续检查
- **警告收集**：非阻断性警告会被收集并返回给调用方
- **线程安全**：RateLimiter 使用 `Mutex` 保证并发安全

### 8. 审计日志

记录所有 Skill 执行和安全事件，支持查询和验证。

#### 记录内容

| 字段 | 说明 |
|------|------|
| `id` | 唯一事件 ID |
| `timestamp` | 事件时间 |
| `user_id` | 用户标识 |
| `skill_name` | Skill 名称 |
| `input_hash` | 输入内容的 SHA-256 哈希 |
| `status` | 执行状态（success/blocked/error） |

#### 使用方法

```
agenthub> /audit                    # 查询所有日志
agenthub> /audit user123            # 按用户查询
agenthub> /audit user123 code.explainer  # 按用户和 Skill 查询
```

#### 实现逻辑

- **隐私保护**：使用 SHA-256 哈希存储输入内容，不存储原文
- **HMAC 签名**：[signed_log.rs](file:///d:/KaKaRoot/AgentHub智能体中枢RustWithQWen/src/audit/signed_log.rs) 支持日志完整性验证
- **文件存储**：审计日志写入本地文件，支持大小限制

### 9. 容错机制

确保系统在部分组件故障时仍能正常运行。

#### 组件

| 组件 | 文件 | 功能 |
|------|------|------|
| **CircuitBreaker** | [circuit_breaker.rs](file:///d:/KaKaRoot/AgentHub智能体中枢RustWithQWen/src/recovery/circuit_breaker.rs) | 熔断器模式，防止级联故障 |
| **Fallback** | [fallback.rs](file:///d:/KaKaRoot/AgentHub智能体中枢RustWithQWen/src/recovery/fallback.rs) | 降级策略，提供备用响应 |
| **ResilientLLM** | [resilient_llm.rs](file:///d:/KaKaRoot/AgentHub智能体中枢RustWithQWen/src/recovery/resilient_llm.rs) | 弹性 LLM 调用 |

#### 实现逻辑

- **熔断器**：失败次数超过阈值时打开熔断器，快速失败
- **自动恢复**：熔断器在半开状态尝试恢复
- **降级策略**：LLM 不可用时返回 Mock 响应

## 配置说明

配置文件位于 `config/agenthub.yaml`，支持以下配置项：

### 服务器配置

```yaml
server:
  host: "127.0.0.1"
  port: 8080
```

### LLM 配置

```yaml
llm:
  default_provider: "mini"
  providers:
    - name: "mini"
      type: openai
      api_key: "your-key"
      model: "model-name"
      api_endpoint: "https://api.example.com/v1"

      timeout_secs: 120
```

```yaml
security:
  max_input_length: 50000
  
  pii_detection:
    enabled: true
    redact_emails: true
    redact_phones: true
    redact_id_cards: true
    redact_ips: false
    redact_api_keys: true
  
  prompt_injection:
    enabled: true
    check_unicode: true
    check_zero_width: true
  
  file_access:
    enabled: true
    workspace_dir: null
    sensitive_files: [".env", ".git/", "id_rsa"]
    allow_symlinks: false
  
  network:
    enabled: true
    allow_external: false
    allowed_domains: []
    block_internal_ips: true
  
  rate_limit:
    enabled: true
    requests_per_minute: 60
    max_burst: 10
  
  output_sanitizer:
    enabled: true
    strip_control_chars: true
    redact_pii: true
```

### 数据流

```
用户输入
  │
  ├─ REPL 模式
  │   │
  │   ├─ 斜杠命令 → Command 解析 → 执行对应操作
  │   └─ 普通文本 → 安全检查 → LLM 调用 → 输出消毒 → Markdown 渲染
  │
  └─ HTTP 模式
      │
      ├─ /api/explain → 安全检查 → Skill 执行 → 输出消毒 → JSON 响应
      └─ /api/test → 安全检查 → Skill 执行 → 输出消毒 → JSON 响应
```

### 技术栈

| 类别 | 技术 |
|------|------|
| 语言 | Rust 2021 Edition |
| 异步运行时 | Tokio |
| HTTP 框架 | Axum |
| CLI | Clap + Rustyline |
| 配置 | Config + Serde YAML |
| LLM 客户端 | Reqwest + Async Trait |
| 代码解析 | Tree-sitter |
| 日志 | Tracing |
| 安全 | Regex + Unicode Normalization |
| 序列化 | Serde + Serde JSON |

## 扩展指南

### 添加新的 LLM 提供商

1. 在 `src/llm/` 下创建新的客户端实现
2. 实现 `LLMClient` trait
3. 在 `MultiLLMClient::create_client()` 中添加匹配分支

### 添加新的 Skill

1. 在 `src/skill/builtins/` 下创建新模块
2. 实现 `SkillExecutor` trait
3. 在 REPL 或 HTTP handler 中注册

### 添加新的 Guardrail

1. 在 `src/guardrails/` 下创建新模块
2. 实现检查逻辑
3. 在 `SecurityPipeline::new()` 中初始化
4. 在 `check_input()` 或 `sanitize_output()` 中调用
